## 多个 server 使用 reuseport 报错(Nginx 1.30)

### 问题描述
在 NGINX 1.30 中，如果多个 server block 在同一 address:port:type 上使用了 reuseport 选项，就会报错 "duplicate listen options"。这个错误是因为 NGINX 1.30 引入了一个新的检查机制，禁止在同一地址上设置重复的 socket 级选项。
```conf
# 存在多个server块，且都使用了reuseport
server{
    listen 443 ssl reuseport;
    listen 443 quic reuseport;
    http2 on;
    # 其他配置
}
```

##### 报错信息
```txt
root@srv1572367:~/apps/NGINX-LUA/000.SOURCE_CODE/NGINX-1.30.0# ./objs/nginx -t -c /root/apps/Berries-NGINX/000.SOURCE_CODE/NGINX-1.24.0/build-output/nginx.conf
nginx: [emerg] duplicate listen options for 0.0.0.0:443 in /root/apps/Berries-NGINX/000.SOURCE_CODE/NGINX-1.24.0/build-output/serviers/cc-manage.nginx.conf:2
nginx: configuration file /root/apps/Berries-NGINX/000.SOURCE_CODE/NGINX-1.24.0/build-output/nginx.conf test failed

```

### 问题处理
```txt

现在完全清楚了。看 NGINX 1.30 源码中关键逻辑：

  // src/http/ngx_http.c:1293-1303
  if (lsopt->set) {
      if (addr[i].opt.set) {
          // "duplicate listen options" 错误!
      }
      addr[i].opt = *lsopt;
  }

  reuseport 会设置 lsopt.set = 1（源码 line 4228）。当第一个 server block 创建了一个带 set=1 的地址条目后，后续任何 server block 只要在同一 address:port:type 上也设置了任何 socket
  级选项（reuseport、bind、backlog 等），就会触发这个错误。

  总结：在 NGINX 1.30 中，reuseport 等 socket 级选项不能跨越多个 server block 使用。


修复方法：把服务器上 /root/apps/Berries-NGINX/000.SOURCE_CODE/NGINX-1.24.0/build-output/serviers/ 下所有 6 个配置文件中的 reuseport 全部删掉：

  listen 443 ssl;
  listen 443 quic;
```
