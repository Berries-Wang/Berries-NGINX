# Q: 为什么多个域名绑定同一个Nginx(IP+Port),通过不同的域名，能访问到不同的应用
> 即： SNI（Server Name Indication） ，TLS 加密协议的补充，它使客户端设备可以指定在 TLS 握手的第一步中尝试访问的域名，从而防止了通用名称不匹配错误
>> SNI 会在“Client Hello（客户端问候）”消息或 TLS 握手的第一步就包含了主机名。（会去匹配server块的server_name，然后使用对应的ssl证书完成握手）


## 相关概念
|名称|介绍|
|-|-|
|- 主机名|- 主机名是连接到网络的设备的名称。在互联网背景中，域名或网站名称是一种主机名。两者都与与域名关联的 IP 地址分开。|
|-|-|
|- 虚拟主机名|- 虚拟主机名是没有自己的IP地址的主机名，它与其他主机名一起托管在服务器上。它是"虚拟的" ，因为它没有专用的物理服务器，就像虚拟现实仅以数字形式存在，而不是在物理世界中一样。|
